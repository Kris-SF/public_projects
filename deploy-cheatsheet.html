Deploy a New Subdomain on Vercel

Repo: Kris-SF/public_projects  |  Last updated: Feb 2026

⚠ ROOT DOMAIN WARNING: moontowermeta.com is WordPress. NEVER add it to any Vercel project. Only add subdomains like xyz.moontowermeta.com.
Step 1 — Tell Claude what you want CLAUDE

You say something like: "Create a new Vite app called cool-tool in the repo"

Claude will:

  • Create the cool-tool/ subdirectory with all source files
  • Add vercel.json inside it (with correct build settings)
  • Commit and push to GitHub

You do nothing. Just wait for Claude to finish.

Step 2 — Create the Vercel project YOU
  1. Go to vercel.com → Add New Project
  2. Import Kris-SF/public_projects (same repo — you can import it multiple times)
  3. Set Root Directory → cool-tool
  4. Framework will auto-detect (Vite, or "Other" for static HTML)
  5. Click Deploy
Step 3 — Add the subdomain in Vercel YOU
  1. In your new Vercel project → Settings → Domains
  2. Type: cool-tool.moontowermeta.com
  3. Click Add
Step 4 — Add the DNS record at your registrar YOU

Add one CNAME record:

Type:  CNAME
Name:  cool-tool
Value: cname.vercel-dns.com

Wait a few minutes for DNS propagation. Done.
